1 Homebrew & core packages

Install Homebrew first, then the core tools everything else depends on.

# Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Apple Silicon: add to PATH (installer will prompt, but for scripts)
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

# Core taps
brew tap davidwhittington/mac-security
brew tap peonping/tap

# Core packages
brew install git gh node antidote starship tmux
brew install btop fastfetch htop tree watch wget
brew install gnupg pinentry-mac git-crypt
brew install mac-security direnv

ohmyzsh/ohmyzsh path:plugins/debian
ohmyzsh/ohmyzsh path:plugins/firewalld
ohmyzsh/ohmyzsh path:plugins/git
ohmyzsh/ohmyzsh path:plugins/history
ohmyzsh/ohmyzsh path:plugins/nvm
ohmyzsh/ohmyzsh path:plugins/podman
ohmyzsh/ohmyzsh path:plugins/sudo

ptavares/zsh-direnv
toku-sa-n/zsh-dot-up
zsh-users/zsh-syntax-highlighting kind:defer
~/.zshrc
eval "$(starship init zsh)"

test -e "${HOME}/.iterm2_shell_integration.zsh" && source "${HOME}/.iterm2_shell_integration.zsh"

source $(brew --prefix)/opt/antidote/share/antidote/antidote.zsh
antidote load

export PATH="$HOME/.local/bin:$PATH"
export GPG_TTY=$(tty)

# direnv
eval "$(direnv hook zsh)"

# Auto-launch tmux in iTerm2 control mode
if [[ -z "$TMUX" && "$TERM_PROGRAM" == "iTerm.app" && -z "$INSIDE_EMACS" && -z "$VSCODE_PID" ]]; then
  tmux -CC new-session -A -s main
fi
~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

3 Starship prompt

Custom Starship config with a two-line layout: git metrics on top, character prompt on the left, directory and toolchain info on the right. Key settings:

SettingValueWhy
command_timeout2000msPrevents git timeout warnings on large repos
ssh_only (hostname)falseAlways shows hostname so you know which machine you're on
git_branch.ignore_branchesmain, masterOnly shows branch name on feature branches
git_metricsenabledShows +added/-deleted on first line
directory.truncation_length2Keeps path short
Config location: ~/.config/starship.toml
Copy from any existing fleet machine. The config is identical across all three. The full file is ~300 lines covering language modules, git status symbols, and battery display.
Key sections to verify
# Top of file
add_newline = true

# Hostname shows locally, not just over SSH
[hostname]

4 Nerd Fonts

JetBrains Mono Nerd Font is the standard across the fleet. The Nerd Font variant includes glyphs for Starship, Powerline, devicons, and other terminal symbols. 

brew install --cask font-jetbrains-mono-nerd-font
SettingValue
iTerm2 fontJetBrainsMonoNFM-Regular 18
PostScript nameJetBrainsMonoNFM-Regular
VariantNerd Font Mono (monospaced glyphs, no overlap)
Not the same as JetBrainsMono-Regular. The non-NF variant looks similar but is missing all the Nerd Font glyphs. Starship symbols will render as boxes or question marks without the NF variant installed.

6 tmux + iTerm2 control mode

tmux sessions auto-launch in iTerm2 control mode. Sessions persist across disconnects. All pane output logs to ~/Documents/logs/ (iCloud-synced). See the full tmux workflow guide for details.

Copy tmux config from existing machine
scp macbook-pro:~/.tmux.conf ~/.tmux.conf
mkdir -p ~/Documents/logs

Key features of the shared ~/.tmux.conf:

7 Git configuration

Performance settings and identity. These make a significant difference for Starship prompt speed.

# Performance (eliminates Starship git timeouts)
git config --global core.untrackedCache true
git config --global core.fsmonitor true
git config --global feature.manyFiles true

# Identity
git config --global user.name "David Whittington"
git config --global user.email "davidwhittington@users.noreply.github.com"
SettingPurpose
core.fsmonitorBackground daemon watches for file changes instead of scanning the whole tree
core.untrackedCacheCaches untracked file results between git status runs
feature.manyFilesEnables preload index and other optimizations for large repos

8 SSH & fleet mesh

All machines in the fleet use ed25519 keys and can SSH to each other without passwords. Generate a key and register it with the fleet. 

# Generate ed25519 key
ssh-keygen -t ed25519 -C "david@$(hostname -s)"

# Copy key to each fleet machine
ssh-copy-id macbook-pro
ssh-copy-id macbook-air
ssh-copy-id mac-pro
~/.ssh/config
Host *
  AddKeysToAgent yes
  UseKeychain yes
  IdentityFile ~/.ssh/id_ed25519
  IdentityFile ~/.ssh/id_rsa

Host macbook-pro
    HostName 192.168.6.0
    User david

Host macbook-air
    HostName 192.168.6.177
    User david

Host mac-pro
    HostName 192.168.5.67
    User david
After adding the key: Ask the fleet admin to add your public key to ~/.ssh/authorized_keys on each existing machine, and add the existing machines' public keys to yours. The fleet should be a full mesh.

9 Security hardening

Run the mac-security audit to check posture, then apply the baseline. 

# Run the audit
mac-security-audit --brief

# Apply SSH hardening
printf '%s\n' \
  'PasswordAuthentication no' \
  'PermitRootLogin no' \
  'PubkeyAuthentication yes' \
  'MaxAuthTries 3' \
  | sudo tee /etc/ssh/sshd_config.d/099-hardening.conf

# Enable stealth firewall
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --setstealthmode on

# Set hostname (replace with actual machine name)
sudo scutil --set HostName "new-machine-name"
CheckExpected
FileVaultEnabled
SIPEnabled
GatekeeperEnabled
Application FirewallEnabled + Stealth Mode
SSH PermitRootLoginno
SSH PasswordAuthenticationno
Remote Management (ARD)Disabled (unless needed)

10 Claude Code

Install globally via npm. Requires Node.js (installed in step 1).

npm install -g @anthropic-ai/claude-code

# Verify
claude --version

Claude Code runs inside tmux sessions. If an SSH connection drops mid-task, the tmux session keeps Claude running. Reattach with tmux -CC attach -s main.
